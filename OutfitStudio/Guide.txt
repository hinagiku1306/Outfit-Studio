5. New/Edit Schedule Menu
A schedule automatically changes your outfit when its conditions are met. Each schedule has a pool of outfits that are picked one by one. Once every outfit has been worn, the pool reshuffles. As a result, you won't see the same outfit twice in a row.
Schedules are checked at the start of each day and every time you change locations. If multiple schedules match at the same time, the one with the highest priority wins. If they have the same priority, one is picked randomly (more on Consistent Tiebreaks in #8. Config Menu).
If you manually apply an outfit, schedules will pause for the rest of the day and resume the next day (more on Lock Manual Outfits in #8. Config Menu).

[screenshot]
For any condition, if nothing is selected, the schedule will match regardless of that condition. For example, leaving Season empty means the schedule works in all seasons.
Schedule Name is auto-generated based on the selected conditions.

Conditions: Season, Weather, Area (Outdoor/Indoor), Location
Special Events: Festival, Wedding
Special Events have the highest priority (marked as Special) and will override any non-special schedule. Note: the outfit is applied when you enter the event, not at the start of the day.

[Total Outfits] -- total number of selected outfits for the schedule

[<number> remaining before reset] -- only available in Edit mode. Total number of outfits in pool before reshuffling.

[Priority] -- determines which schedule will be picked when multiple schedules match.
Value: High/Medium/Low
Recommendation: for generic rules (e.g. Season=Spring), set Low Priority. For more specific rules (e.g. Weather=Rainy, Area=Outdoor), set High Priority for a better chance of getting picked.

[Rotation] -- determines how often the schedule picks a new outfit from its pool.
Daily -- one outfit is picked the first time the schedule triggers. That same outfit stays on for the rest of the day no matter how many more times the schedule triggers.
On Warp -- a new outfit is picked every time you change locations and this schedule triggers.
Example: A Spring schedule with 3 outfits: X, Y, Z. With Daily, if X is picked on the first trigger, you'll wear X for the rest of the day. With On Warp, you'll cycle through X, Y, Z as you move between areas.
TLDR: On Warp = frequent outfit changes. Daily = one look per day.

8. Config Menu
[screenshot]
I will leave out the more obvious ones.

Include Hair in Outfit Sets=true
Enables saving hair with outfit sets. Note: with this enabled, changing your hair in the menu also counts as manually applying an outfit, which may pause schedules (see Lock Manual Outfits below).

Consistent Tiebreaks=true
When multiple schedules with the same priority compete (e.g. Season=Spring and Area=Outdoor both at Medium), the winner is normally picked randomly each time. With this enabled, the winning schedule stays the same every time those schedules compete, instead of being randomly re-rolled. Enable this if you want predictable results.

Lock Manual Outfits=true
When enabled, manually applying an outfit pauses ALL schedules for the rest of the day. When disabled, schedules only pause at that specific location.
